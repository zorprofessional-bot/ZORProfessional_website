Now add Supabase as the content and lead backend for the ZOR Professional website.

Important:
The public website has been refactored into a no-scroll deck-style presentation interface.

Do not go back to scroll-based page sections.

The content model must follow the new concept:

* Main navigation items are “chapters”
* Each chapter contains multiple “slides”
* Public pages are deck-style screens, not long scrolling pages
* Admin should later be able to edit chapters and slides
* Blog article pages may have an internal scrollable reader panel
* Admin pages may use normal scrolling

Do not redesign the public website.
Do not break the DeckShell, DeckPage, DeckSlide, DeckControls, SlideIndicator or chapter theme system.
Follow PROJECT_BRIEF.md, DESIGN_SYSTEM.md, CONTENT_STRUCTURE.md and DECK_ARCHITECTURE.md.

Use Supabase for:

* deck chapters
* deck slides
* products
* blog posts
* leads
* career positions
* career applications
* site settings
* image/file storage
* admin authentication

Do not use the old “page_sections” model.
If any previous code references page_sections, replace it with deck_chapters and deck_slides.

* public can read active deck_chapters
* public can read active deck_slides
